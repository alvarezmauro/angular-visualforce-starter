Visualforce Remote Action Demo

Click on a button to perform a visualforce remote action.

Actions are defined in config/vfrActions.constant.js. This allows us to define and configure all remote actions in one place. Each action has a key, an action name and options. The key is used to reference an action, the action name and options are passed to the invokeAction function.

  • The key does not need to match the action name, but it is helpful to be consistent.
  • Action name needs to be prefixed with a controller class. This can be any controller referenced on your visualforce page.
  • We provide some default options used by all actions, but these can be overridden on a per action basis.
For more information on Visualforce Remote Actions see JavaScript Remoting for Apex Controllers.

Since we're not hosting this page in Salesforce we need to mock out the actions. Actions are mocked in mocks/mockedVfrActions.js. vfrMock can be configured for latency and error using the vfrMockConfig object in index.html. This allows you to test how your program functions when things go badly.
